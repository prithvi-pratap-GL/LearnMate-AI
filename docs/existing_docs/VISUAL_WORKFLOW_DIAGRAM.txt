┃         │                                                                    ┃
┃         └──→ question_service.generate_questions(topic, "beginner")        ┃
┃              │                                                              ┃
┃              └──→ Create QUESTION_GENERATION_PROMPT                        ┃
┃                   "Generate 5 core concept MCQs about {topic}..."         ┃
┃                   │                                                         ┃
┃                   └──→ query_model(prompt, model="Mistral-7B")            ┃
┃                        │                                                    ┃

┃                        │  Hugging Face Inference API         │            ┃
┃                        │  POST /models/mistralai/Mistral-7B  │            ┃
┃                        │  Input: {"inputs": prompt}          │            ┃
┃                        └─────────────┬───────────────────────┘            ┃
┃                                      │                                     ┃
┃                    Response: [{"generated_text": "[{...}, {...}, ...]"}]  ┃
┃                                      │                                     ┃
┃                                      ↓                                     ┃
┃                    Parse JSON array of 5 MCQs                             ┃
┃                                      │                                     ┃
┃                                      ↓                                     ┃
┃              ┌──────────────────────────────────────┐                     ┃
┃              │ Response to Frontend:                 │                     ┃
┃              │ {                                     │                     ┃
┃              │   "questions": [                      │                     ┃
┃              │     {                                 │                     ┃
┃              │       "question": "What is...",       │                     ┃

┃              │       "correct_answer": "A"           │                     ┃
┃              │     },                                │                     ┃
┃              │     ... (4 more)                      │                     ┃
┃              │   ],                                  │                     ┃
┃              │   "round": 1,                         │                     ┃
┃              │   "total_questions": 5                │                     ┃
┃              │ }                                     │                     ┃
┃              └──────────────────────────────────────┘                     ┃
┃                                      │                                     ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

┃  │                                                                          ┃

┃         "student_name": "Alice",                                           ┃

┃             "question": "What does ML stand for?",                         ┃

┃ STAGE 4: EVALUATION (LLM AS JUDGE)                                          ┃
┣━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┫
┃                                                                              ┃
┃  Backend (learning.py)                                                      ┃
┃         │                                                                    ┃
┃         └──→ evaluation_service.evaluate_learning(request)                ┃
┃              │                                                              ┃
┃              ├─→ STEP 1: Calculate Score                                  ┃
┃              │   Correct answers: 3/5 = 60%                               ┃
┃              │                                                              ┃
┃              ├─→ STEP 2: Create EVALUATION_PROMPT                         ┃
┃              │   "Act as expert evaluator for topic: Python Programming"  ┃
┃              │   "Questions and Answers: {all answers}"                   ┃
┃              │                                                              ┃
┃              └─→ STEP 3: Query Specialized Judge Model                    ┃
┃                  query_model(prompt, model="Llama-2-70b-chat-hf")        ┃
┃                  │                                                         ┃
┃                  ┌─────────────────────────────────────┐                 ┃
┃                  │  Hugging Face Inference API         │                 ┃
┃                  │  POST /models/meta-llama/Llama-...  │                 ┃
┃                  │  (High-capacity evaluation model)   │                 ┃
┃                  └─────────────┬───────────────────────┘                 ┃
┃                                │                                          ┃
┃          Llama 70B Analyzes:                                             ┃
┃          ├─→ Correctness of each answer                                  ┃
┃          ├─→ Depth of understanding                                      ┃
┃          ├─→ Key strengths demonstrated                                  ┃
┃          ├─→ Areas needing improvement                                   ┃
┃          └─→ Appropriate proficiency level                               ┃
┃                                │                                          ┃
┃                                ↓                                          ┃
┃          Response:                                                        ┃
┃          {                                                                ┃
┃            "score": 60,                                                   ┃
┃            "strengths": ["Good understanding", "Practical awareness"],   ┃
┃            "weak_areas": ["Advanced topics", "Edge cases"],              ┃
┃            "level": "Intermediate"                                       ┃
┃          }                                                                ┃
┃                                │                                          ┃
┃                                ↓                                          ┃
┃              ┌──────────────────────────────────────┐                   ┃
┃              │  DECISION POINT                       │                   ┃
┃              │  if score < 50%:                      │                   ┃
┃              │    └─→ Show Performance Analysis      │                   ┃
┃              │        and END Assessment             │                   ┃
┃              │  else (score >= 50%):                 │                   ┃
┃              │    └─→ PROCEED TO ROUND 2  ✓          │                   ┃
┃              └──────────────────────────────────────┘                   ┃
┃                                │                                          ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

┌─ MODEL SELECTION STRATEGY ────────────────────────────────────────────────────┐
│                                                                               │
│  Question Generation    → Mistral 7B                                         │
│  ├─ Why: Fast, efficient, sufficient for MCQ creation                       │
│  ├─ Speed: ⚡ Fast (5-10 seconds)                                           │
│  └─ Quality: ✅ Good for structured output                                  │
│                                                                               │
│  Evaluation/Judging     → Llama 70B  ← Uses different model!                │
│  ├─ Why: High-capacity, nuanced analysis                                    │
│  ├─ Speed: 🐌 Slower (15-30 seconds)                                        │
│  ├─ Quality: ⭐⭐⭐ Excellent for judgment                                  │
│  └─ Fallbacks: Mistral Large → Nous Hermes → Mistral 7B                    │
│                                                                               │
└───────────────────────────────────────────────────────────────────────────────┘

┌─ QUESTION UNIQUENESS GUARANTEE ───────────────────────────────────────────────┐
│                                                                               │
│  Problem:   Student sees same question in Round 1 and Round 2                │
│  Solution:  Smart filtering before returning Round 2 questions              │
│                                                                               │
│  Process:                                                                     │
│  1. Generate 5+ advanced questions                                           │
│  2. Extract all Round 1 question texts (case-insensitive)                   │
