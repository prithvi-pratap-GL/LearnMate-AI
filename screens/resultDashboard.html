Performance Analysis

Your performance across both rounds shows a solid foundational understanding of Python with clear room to grow in advanced concepts. Here's a detailed breakdown.

Question 1 — Data Types

You correctly identified that type([]) returns <class 'list'>. This shows a good understanding of Python's type system. One thing to explore further: Python's type() function always returns the metaclass, which is why the output includes the class keyword.

Question 2 — Function Definition

Correct. Python uses the def keyword, unlike JavaScript (function) or Kotlin (fun). The distinction matters when reading open-source code across different languages.

Question 3 — String Length

Correct. len() returns the number of Unicode characters, not bytes. This becomes important when working with multi-byte encodings (e.g., emoji, Chinese characters) — one character can be multiple bytes in UTF-8.

Question 4 — Immutability

Correct. Tuples are immutable; lists, dicts, and sets are mutable. Immutability enables tuples to be used as dictionary keys and set members — a property that lists cannot provide.

Question 5 — Virtual Environment

You selected the wrong option. The correct command is python -m venv env. The -m flag runs a module as a script. Common mistake: using pip create which doesn't exist, or using virtualenv (a separate package, not built into Python 3 stdlib).
